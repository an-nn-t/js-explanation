1JavaScriptとは？

JavaScriptは、ウェブページに動的な機能を追加するためのプログラミング言語です。もともとはブラウザ上で動作するスクリプト言語として開発されましたが、現在ではNode.jsなどの登場により、サーバーサイド開発にも用いられるなど、その活用範囲は非常に広くなっています。

JavaScriptの主な特徴

  • 動的型付け言語: 変数の型を実行時に決定します。柔軟性が高い反面、大規模開発では意図しないエラーの原因になることもあります。
  • プロトタイプベースのオブジェクト指向: クラスベースの言語とは異なり、プロトタイプを元にオブジェクトを生成します。
  • イベント駆動型: ユーザーのマウス操作やキーボード入力といった「イベント」をきっかけに処理を実行するスタイルを得意とします。
  • 豊富なエコシステム: npm (Node Package Manager) を通じて、世界中の開発者が作成した膨大な数のライブラリやフレームワークを利用できます。

基本構文の紹介

いくつかの基本的な構文を見ていきましょう。

変数宣言

ES2015以降、let（再代入可能）とconst（再代入不可）の使用が推奨されています。

// 変数 (再代入可能)
let message = "こんにちは";
message = "こんばんは";

// 定数 (再代入不可)
const siteName = "JS Master";

制御構文 (if文 / for文)

プログラムの流れを制御します。

const score = 85;

if (score >= 80) {
  console.log("素晴らしい！合格です。");
} else {
  console.log("あと少し！再挑戦しましょう。");
}

for (let i = 0; i < 3; i++) {
  console.log(`カウント: ${i}`);
}

関数

一連の処理をまとめて名前をつけたものです。

// 関数の定義
function greet(name) {
  return `こんにちは、${name}さん！`;
}

// 関数の呼び出し
const greeting = greet("山田");
console.log(greeting); // "こんにちは、山田さん！"

より詳しい文法や使い方については、MDN Web Docs が非常に信頼性の高いリソースです。

2TypeScript: 静的型付けの恩恵

TypeScriptは、Microsoftによって開発された、JavaScriptに静的型付けの機能を追加したスーパーセット（上位互換）言語です。TypeScriptで書かれたコードは、最終的にJavaScriptにコンパイル（変換）されて実行されます。

なぜTypeScriptを使うのか？

  1. コードの品質向上: 変数や関数の引数・戻り値に「型」を明示することで、開発段階（コードを書いている時）にエラーを発見しやすくなります。
  2. 開発効率の向上: エディタのコード補完や型情報ヒントが強力に機能し、大規模なアプリケーションでも安全かつ効率的に開発を進められます。
  3. 可読性と保守性の向上: コードに型が記述されていることで、他の開発者がコードの意図を理解しやすくなります。

JavaScriptとの比較

以下のコードは、TypeScriptで型を定義する簡単な例です。: stringや: numberといった部分が型注釈です。これにより、意図しない型のデータが代入されることを防ぎます。 

// TypeScriptでは型を明記する
let userName: string = "鈴木";
let userAge: number = 30;

// 間違った型の値を代入しようとすると、コンパイル時にエラーが発生する
// userAge = "三十歳"; // Error! Type 'string' is not assignable to type 'number'.

function add(a: number, b: number): number {
    return a + b;
}

TypeScriptの導入は、特に中規模以上のプロジェクトにおいて、バグの少ない堅牢なアプリケーション開発を力強くサポートします。詳細はTypeScript公式ドキュメントで確認できます。

3React: コンポーネントベースのUI開発

Reactは、Facebook（現Meta）社が開発した、ユーザーインターフェース（UI）を構築するためのJavaScriptライブラリです。宣言的なUIとコンポーネントベースの設計が特徴で、現在のフロントエンド開発で最も人気のある技術の一つです。

Reactの核心的な考え方

  • コンポーネント指向: UIを「コンポーネント」と呼ばれる独立した部品の組み合わせとして構築します。ヘッダー、ボタン、フォームなどをそれぞれ部品として作成し、それらを組み合わせてページ全体を作り上げます。これにより、コードの再利用性が高まり、管理が容易になります。
  • 宣言的なView: 「もしデータがこの状態なら、UIはこのように表示される」というように、状態に応じたUIの見た目を記述します。データの状態が変わると、Reactが効率的に差分だけを検知してDOMを更新してくれるため、開発者はUIの状態管理に集中できます。
  • JSX: JavaScriptの拡張構文で、JavaScriptコードの中にHTMLのようなタグを直接記述できます。これにより、UIの構造が直感的に理解しやすくなります。

Reactコンポーネントの例 (JSX)

以下のコードは、クリックするとカウントが増える簡単なカウンターコンポーネントの例です。 

import React, { useState } from 'react';

function Counter() {
  // `useState` は状態を管理するためのフック(Hook)
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>現在のカウント: {count}</p>
      <button onClick={() => setCount(count + 1)}>
        カウントアップ
      </button>
    </div>
  );
}

このように、UIの見た目とロジックをひとまとまりのコンポーネントとして管理することで、複雑なアプリケーションも整理された状態に保つことができます。
