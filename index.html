変数宣言: `const`, `let`, `var`

`const`は再代入不可能な定数、`let`は再代入可能な変数を宣言します。どちらもブロックスコープを持ち、意図しない変数の上書きを防ぎます。`var`は関数スコープを持つ古い形式で、現在は推奨されません。

// ES2015以降の推奨
const PI = 3.14; // 再代入不可
let userAge = 25;   // 再代入可能
userAge = 26;     // OK

function checkScope() {
  if (true) {
    let blockVar = "Visible";
    const blockConst = "Also visible";
  }
  // console.log(blockVar); // ReferenceError
}

アロー関数と `this`

アロー関数は `function` キーワードよりも簡潔に記述でき、自身の `this` を持ちません。そのため、アロー関数内の `this` は、それが定義された外側のスコープの `this` を参照します。これにより、クラスやコールバック関数内での `this` の扱いが直感的になります。

function TraditionalPerson() {
  this.age = 0;
  setInterval(function growUp() {
    // この中の `this` はグローバルオブジェクトを指す(非strictモード)
    // this.age++; // うまく動かない
  }, 1000);
}

function ModernPerson() {
  this.age = 0;
  setInterval(() => {
    // アロー関数は外側の`this`(ModernPersonのインスタンス)を指す
    this.age++; // 正常に動作
  }, 1000);
}

ExpressによるREST API構築

ExpressはNode.jsで最も人気のあるWebフレームワークです。ルーティング、ミドルウェア、リクエスト・レスポンスの処理を容易にし、堅牢なAPIを迅速に開発できます。

// 必要なパッケージをインストール: npm install express
const express = require('express');
const app = express();
const port = 3000;

// JSONリクエストボディを解析するミドルウェア
app.use(express.json());

// GETリクエストへの応答
app.get('/api/users', (req, res) => {
  res.json([
    { id: 1, name: 'Taro' },
    { id: 2, name: 'Hanako' }
  ]);
});

// POSTリクエストへの応答
app.post('/api/users', (req, res) => {
  const newUser = req.body;
  console.log('New user added:', newUser);
  res.status(201).json(newUser);
});

app.listen(port, () => {
  console.log(`Server running at http://localhost:${port}`);
});

インターフェース (Interface)

オブジェクトが持つべき構造（プロパティやメソッドの型）を定義する機能です。これにより、特定の構造を持つオブジェクトを強制でき、コードの意図が明確になります。

interface User {
  id: number;
  name: string;
  email?: string; // `?` はオプショナルなプロパティ
  getInfo(): string;
}

const userA: User = {
  id: 101,
  name: "Kenji",
  getInfo() {
    return `ID: ${this.id}, Name: ${this.name}`;
  }
};

ジェネリクス (Generics)

ジェネリクスは、特定の型に依存せず、再利用可能なコンポーネント（関数やクラス）を作成するための機能です。使用する際に型を指定することで、型安全性を保ちながら柔軟なコードを書けます。

// Tは型引数(Type Parameter)
function getFirstElement(arr: T[]): T | undefined {
  return arr[0];
}

const numbers = [1, 2, 3];
const firstNumber = getFirstElement(numbers); // firstNumberの型は `number`

const strings = ["a", "b", "c"];
const firstString = getFirstElement(strings); // firstStringの型は `string`

コンポーネント指向のUI開発を牽引するReactとVue。それぞれの思想、仮想DOMの仕組み、状態管理の基本を比較しながら、実践的な使い方を解説します。

25分

React: 宣言的UIと仮想DOM

ReactはUIを「状態の関数」として捉えます。状態(state)が変わると、Reactが効率的に差分を計算（仮想DOMを利用）し、実際のDOMに必要な変更のみを適用します。これにより、パフォーマンスの高いUIが実現されます。

// 親から子へデータを渡す `props`
function Welcome(props) {
  return <h1>こんにちは、{props.name}さん！</h1>;
}

// 自身の状態を管理する `useState`
import { useState } from 'react';

function Counter() {
  const [count, setCount] = useState(0);
  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => setCount(count + 1)}>
        Increase
      </button>
    </div>
  );
}

Vue.js: リアクティブなデータバインディング

VueはデータとDOMを「リアクティブ」に結びつけます。JavaScriptオブジェクトの状態を変更するだけで、関連するDOMが自動的に更新されるのが大きな特徴です。シンプルで学習しやすい文法も魅力です。

<!-- Vue Template -->
<div id="app">
  <p>{{ message }}</p>
  <p>Count: {{ count }}</p>
  <button @click="increment">Increase</button>
</div>

<!-- Vue Script (Composition API) -->
<script>
  import { ref, computed } from 'vue';

  export default {
    setup() {
      const count = ref(0);
      const message = computed(() => `現在のカウントは${count.value}です。`);

      function increment() {
        count.value++;
      }

      return { count, message, increment };
    }
  };
</script>
